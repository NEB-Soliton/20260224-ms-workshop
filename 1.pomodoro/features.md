# Pomodoroタイマー 実装機能一覧（MVPベース）

## 1. 必須機能（MVP）

### 1.1 タイマー制御
- 開始
- 一時停止
- 再開
- リセット
- 状態管理（idle / running / paused / completed）

### 1.2 モード管理
- 作業（focus）
- 短休憩（short_break）
- 長休憩（long_break）
- セッション完了時の次モード遷移（MVPは固定ルールで可）

### 1.3 タイマー表示UI
- 残り時間表示（MM:SS）
- 円形プログレス表示
- 現在モード表示（例：作業中）

### 1.4 今日の進捗表示
- 完了ポモドーロ数
- 当日の累計集中時間

### 1.5 API（設定・履歴）
- GET /api/settings（設定取得）
- PUT /api/settings（設定更新）
- POST /api/sessions（完了セッション記録）
- GET /api/sessions?date=YYYY-MM-DD（指定日の履歴/集計取得）

---

## 2. 品質・運用上、実装が必要な機能

### 2.1 タイマー精度
- 残時間は「終了予定時刻 - 現在時刻」で計算
- 秒の単純減算によるドリフトを抑制

### 2.2 永続化
- 設定と履歴を保存（初期はJSON）
- ページ再読み込み後の復元

### 2.3 バリデーションとエラーハンドリング
- 設定更新APIの入力バリデーション
- エラーレスポンス形式の統一
- フロント側でのエラー表示分岐

### 2.4 テスト
- 状態遷移ロジックの単体テスト
- API契約テスト（正常系・異常系）
- サービス層の単体テスト

### 2.5 責務分離（保守性）
- タイマー計算ロジックとDOM更新ロジックを分離
- Flaskルートは薄く保ち、業務ロジックはサービス層へ集約
